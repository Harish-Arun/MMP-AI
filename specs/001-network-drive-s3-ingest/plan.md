# Implementation Plan: 001 Network Drive S3 Ingest

**Branch**: `001-network-drive-s3-ingest` | **Date**: 2026-03-10 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/001-network-drive-s3-ingest/spec.md`

## Summary

Standalone Python 3.11 microservice that SFTP-polls a Windows network drive, uploads new `.pdf` files to S3 with SHA-256 hashing and filename-based deduplication (backed by MongoDB), and triggers the mmp-ai-engine workflow via an S3 → Lambda → REST push bridge. Runs as a single Docker container. LocalStack free tier (`SERVICES=s3,lambda`) covers local development. Full test pyramid: unit (`pytest` + `moto` + `mongomock-motor`), BDD (`pytest-bdd` with Gherkin from spec acceptance scenarios), and integration (real LocalStack + MongoDB).

## Technical Context

**Language/Version**: Python 3.11  
**Primary Dependencies**: `asyncssh` (SFTP), `boto3` (S3), `motor` (MongoDB async driver), `tenacity` (retry), `pydantic-settings` + `PyYAML` (config), `structlog` (JSON logging), `opentelemetry-sdk`, `prometheus-client`  
**Storage**: MongoDB (`motor`) — `upload_records` collection; unique index on `filename`  
**Testing**: `pytest`, `pytest-asyncio`, `pytest-bdd`, `moto[s3]`, `mongomock-motor`, `pytest-cov`  
**Target Platform**: Linux Docker container (python:3.11-slim); LocalStack on developer workstation  
**Project Type**: background-service (daemon poll loop + lightweight health HTTP server)  
**Performance Goals**: ≤ 60s from file write-complete to S3 upload confirmed (SC-001); handles ≥ 50 simultaneous file arrivals without loss (SC-003)  
**Constraints**: No partial file uploads (FR-003); zero duplicate workflow triggers (SC-004); graceful SFTP reconnect within one poll interval (SC-006)  
**Scale/Scope**: Single-instance service; expected volume ≤ 100 documents/day; single `upload_records` MongoDB collection

### Phase 2 — SFTP Watcher *(parallel with Phase 3 & 4)*

**Step 6**: Write `src/sftp/watcher.py` — `SFTPWatcher` class:
- `__init__(settings, record_store)` — inject settings and store
- `async def start()` — main loop: `while True: await self._poll(); await asyncio.sleep(settings.sftp_poll_interval_s)`
- `async def _poll()` — connect (or reuse connection), list remote dir, filter by extension allowlist, cross-reference RecordStore
  - **New file**: create `FileEvent`, store in `_pending` dict
  - **Known file in `_pending`**: compare `current_size` vs `size_at_previous_poll`
    - Same + non-zero → `write_complete=True` → yield to uploader
    - Changed → update size, continue waiting
    - Zero across 2 polls → WARNING log, remove from `_pending`
  - **Known file in RecordStore**: WARNING log "duplicate filename", skip
- `async def _connect_with_retry()` — `tenacity` on `asyncssh.connect()` with exponential backoff; structured ERROR log on final failure
- All transitions logged as structured JSON via `structlog` (FR-009)

### Phase 3 — S3 Uploader *(parallel with Phase 2 & 4)*

**Step 7**: Write `src/uploader/s3_uploader.py` — `S3Uploader` class:
- `__init__(settings)` — create `boto3.client('s3', endpoint_url=settings.s3_endpoint_url or None, region_name=settings.aws_region)`
- `async def upload(file_event: FileEvent, sftp_conn) -> UploadRecord` — decorated with `@retry(wait=wait_exponential(...), stop=stop_after_attempt(settings.max_retries), before_sleep=before_sleep_log(...))`
  - Stream-read file bytes from SFTP conn
  - Compute SHA-256 digest while reading (streaming, not load-all-in-memory)
  - `s3.put_object(Bucket=..., Key=f"{key_prefix}/{filename}", Body=bytes, Metadata={"sha256": hash, "detection_timestamp": ..., "original_filename": filename})`
  - Return populated `UploadRecord(status=SUCCESS, sha256_hash=..., upload_timestamp=now)`
- On exhausted retries: raise `UploadFailedError` → caller logs structured ERROR with `retry_count`, `failure_reason` (FR-010)
- `endpoint_url=None` → real AWS; `endpoint_url=http://localhost:4566` → LocalStack (zero code branching)

### Phase 5 — Lambda Bridge

**Step 9**: Write `lambda/handler.py`:
```python
import os, json, hashlib
import httpx, structlog
from tenacity import retry, wait_exponential, stop_after_attempt

ENGINE_REST_URL = os.environ["ENGINE_REST_URL"]
MAX_RETRIES = int(os.environ.get("MAX_RETRIES", 3))
REQUEST_TIMEOUT_S = int(os.environ.get("REQUEST_TIMEOUT_S", 10))

@retry(wait=wait_exponential(multiplier=2, min=1, max=30), stop=stop_after_attempt(MAX_RETRIES))
def _post_to_engine(payload: dict):
    resp = httpx.post(f"{ENGINE_REST_URL}/api/v1/workflows/trigger", json=payload, timeout=REQUEST_TIMEOUT_S)
    if resp.status_code in (400, 409):
        log.warning("non_retryable_engine_response", status=resp.status_code)
        return  # do not raise — non-retryable
    resp.raise_for_status()  # 5xx raises → tenacity retries

def lambda_handler(event, context):
    record = event["Records"][0]["s3"]
    bucket = record["bucket"]["name"]
    key = record["object"]["key"]
    size = record["object"]["size"]
    # Retrieve sha256 + detection_timestamp from S3 object metadata
    ...
    _post_to_engine({"s3_bucket": bucket, "s3_key": key, "filename": filename, ...})
```
- Non-retryable `400`/`409` → log WARNING, return (do not raise → no DLQ)
- `5xx` / timeout → `raise_for_status()` → tenacity retries → on exhaustion raises → Lambda invocation fails → DLQ captures original S3 event
- Write `lambda/requirements.txt`: `httpx>=0.26.0`, `tenacity>=8.2.0`, `structlog>=24.1.0`
